# Ludots Doc Governance

Use this skill when creating or updating Ludots documentation, fixing broken links, or checking SSOT integrity.

## Load

- `references/doc-governance-checklist.md`
- `references/link-validation.md`
- `references/report-template.md`

## Rules

- Keep rules in SSOT documents, not in audits or temporary notes.
- Validate markdown links and repository-relative code paths.
- Link every claim to concrete `docs/...`, `src/...`, `scripts/...`, or `tests/...` paths.

## Outputs

- `artifacts/doc-governance-report.md`
- optional `artifacts/doc-governance-missing-paths.tsv`
